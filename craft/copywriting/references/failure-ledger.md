# Failure Ledger

Root cause: verification treated as a feature of *some* claims instead of the default for
*all* claims — rigorous where explicit (stats, verbatims), corner-cutting everywhere it was
implied. Seven-plus corrections, one cause. This ledger names the faces that one failure
wore, so the next draft catches them before delivery.

Source: Every Agent messaging framework session, July 13, 2026 (product referred to as
"@Every" / "the Every Agent," formerly Plus One). The rules that prevent these live in
`references/messaging-doc-discipline.md` (document discipline) and
`references/copywriting-line-rules.md` (line-level, C1–C5).

---

## The faces it wore

- Scaffolding prose presented as structure ("three buyers, one thread").
- Architecture cited by label instead of cashed in as messages.
- Personas written from plausibility, then defended as if researched.
- Evidence verified in one section, dropped when writing the next.
- Internal escalation ("pending Austin") where public research could answer.
- Bare citations ("Panopto") after the full-citation standard was long established.
- A competitor's setup instruction quoted as their messaging, which hid that they'd started claiming our territory.
- A B2B sales-motion cliché ("reached through the Operator's case, almost never cold") imported from enterprise SaaS folklore and stated as a finding — probably backwards for Every's funnel, where founders discover tools directly; sales-motion/buying-journey claims carry [H] until install data confirms (the "Builder installs → Operator feels → Founder approves" motion is tagged accordingly).
- Imitation compression (Apple/Palantir surface register without a load-bearing fact; see C1–C5).
- Broken grammar shipped twice under style pressure (pronouns with no antecedent; a line whose premise defeated its own twist) — rhythm checked, literal meaning not.
- Same-sounding lines across archetypes/pillars — argument compressed into every sentence (claim + mechanism + differentiation) producing uniform strategy-voice instead of copy; one job per line.
- Reacting instead of executing: offering process commentary/canonical analysis/unrequested next steps in place of the deliverable asked for — when the principal specifies the deliverable, produce it; hold commentary unless asked.

## The test for any draft

For every sentence, ask "how do I know this?" If the answer is "it sounds right," the
sentence is either researched before shipping or tagged as hypothesis. No third option.
