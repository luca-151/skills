What the atom supports today
  • Status bar with optional Focus-mode moon glyph (statusBar.dnd: true) and arbitrary statusBar.time
  • Three header styles: model-tag (e.g. ChatGPT 5.1 ▾), title-only (e.g. ChatGPT ›), plain-title (e.g. Get Plus ✦)
  • Configurable header right-icons: personPlus, dottedCircle, edit, more
  • User text bubbles (right-aligned, #F4F4F4, 28px radius) and user image attachments (same radius, square aspect option)
  • Assistant prose with markdown-ish formatting: bold, italic, H1/H2/H3, ordered + bullet lists, horizontal rule, citation chip [[cite:Source]], inline emoji prefix [[icon:💡]]
  • Optional OpenAI spiral logo + bold response title row
  • Thumbs-up / thumbs-down feedback chips under each assistant message (suppress with feedback: false)
  • Empty state (no messages) with centered spiral mark
  • Composer pill with placeholder, typed text + blinking caret, black active state, and black streaming stop-square state
  • Optional Apps-SDK GPT chip in the composer (composer.chip.name)
  • Fixed 9:16 viewport at 750×1334 (DPR 3 → 2250×4002 PNG)
Animation v1 — typing + send + stream
Molecule: create-chatgpt-video-ad

Canonical example: "Hey what's happening in this app?"

End-to-end demo of the molecule, driving the chatgpt-mockup atom + the new render-ios-keyboard atom from a 15-event timeline. One continuous Playwright recording (no scene cuts), ~12.5s. Subtle SFX overlay: key-tap per word during typing, one send-tap on send, stream-tick every 12 streamed words, response-done at the end. No music bed by default.

Verified beats (via /watch QC):

  • t=0 — keyboard up, empty composer, "ChatGPT" header with personPlus + dotted-circle
  • t=0.6–3.0 — typing animation, caret visible, send button activates as soon as first char appears
  • t=3.2 — one beat: bubble pops + keyboard slides down + header swaps to edit+more + send becomes black stop-square
  • t=3.55–4.10 — single gray loading dot at the left gutter (NOT three pulsing dots; that's iMessage)
  • t=4.10+ — assistant response streams word-by-word with a 220ms opacity+blur ramp per word
  • t=10.8 — send button returns to gray-disabled, response complete
Out of scope for v1 (called out in SKILL.md)
  • Dark mode (light mode only per your spec)
  • Horizontal-scroll carousel cards inside an Apps-SDK response (the SeatGeek 4-up shows tickets/seat-maps in a horizontal scroller)
  • On-screen iOS keyboard chrome (composer only)
  • Markdown tables, fenced code blocks with syntax highlighting, nested lists
  • Memory Off / chip-row UI element below the header (could be added as a separate thread.chips field if needed)
