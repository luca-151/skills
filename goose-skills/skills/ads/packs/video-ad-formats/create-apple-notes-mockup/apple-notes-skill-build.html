1 · Title only — cursor on title

examples/frame-1-title-only.json

First frame of the recording: the user has just typed "Hello" and the yellow caret blinks at the end of the title. Lowercase QWERTY keyboard with autocomplete suggestions Helloo / Hello's / Hellooo.

What to confirm

  • "Hello" rendered as bold display headline at the same size + weight.
  • Yellow caret (#FFCC00) sits right after the "o" with no extra padding.
  • Lowercase QWERTY (shift not held).
  • Suggestion strip reads Helloo · Hello's · Hellooo — and the apostrophe is curly.
  • Format-toolbar pill above the keys shows Aa / checklist / table / paperclip / pen / AI atom.

2 · Mid-typing — two paragraphs, cursor on new line

examples/mid.json

Canonical "this is a screen recording" state. The first body paragraph wraps after "typing on" → "apple notes app." on the next line. An empty paragraph below holds the yellow caret on its own line. Upper-case QWERTY (shift held). Low battery, red.

What to confirm

  • Body wraps at the same word: typing on ends line 1.
  • Empty paragraph below has visible vertical room for the caret.
  • Battery icon is red (low-battery state).
  • No Dynamic Island — the top status bar is clean (time left, signal/wifi/battery right).
  • Yellow caret aligned to the body left margin, not the title margin.

3 · Long note — autocorrect underlines, no keyboard

examples/long.json

Multi-paragraph note in viewing/scroll state (keyboard dismissed). Demonstrates autocorrect underlines (only lowercase this and how are flagged — capital This is not), smart-quote handling on Let's and I'm, and the body's true paragraph-gap spacing.

What to confirm

  • Yellow underline under how, this (lowercase) but NOT under This at the start of paragraph 1.
  • Let's and I'm use curly apostrophes ', not straight.
  • Paragraph-to-paragraph spacing reads as one empty visual line (no doubled spacing).
  • Caret at the end of the last paragraph.
  • Battery red (battery_low: true).

4 · Note with checklist

examples/with-checklist.json

Demonstrates the checklist block: filled yellow circles with white checks + gray strikethrough on completed items, gray-outlined empty circles on incomplete items. No reference frame from the typing video for this state — included to prove the block renders end-to-end.

What to confirm

  • Checked items have yellow-filled circle + white check, strikethrough on text.
  • Unchecked items have gray outline circle, regular text.
  • Checkbox vertically aligned with text baseline (no floating high or low).
  • Smart quote on "Joe's", "there's".

5 · Note with embedded image + caption

examples/with-image.json

Demonstrates the image block: rounded corners ~18px, gray caption beneath. Relative paths in the spec are resolved against the JSON's directory and inlined as base64 by render.js so the output HTML is self-contained.
